Observable Plugin Guidance (Node.js)

Goal:
- Create an observable-* plugin for logging/metrics/tracing backends.
- Node.js plugins are ESM-only. Use `"type": "module"`, `NodeNext`, and `.js` specifiers for relative imports.

Steps:
1. Create plugin folder: src/plugins/observable-<name>.
2. Create index.ts with Config + Plugin (BSBObservable).
   - Config defines plugin metadata and config schema (only include fields you actually have).
   - Implement logging/metrics/tracing methods you support.
   Example index.ts:
   ```ts
   import { BSBObservable, BSBObservableConstructor, createConfigSchema, BSBError } from "@bsb/base";
   import * as av from "anyvali";

   const Config = createConfigSchema(
     {
       name: "observable-example",
       description: "Example observable plugin",
       tags: ["observable", "example"],
       documentation: ["./docs/observable-example.md"]
     },
     av.object({
       level: av.string().default("info")
     }, { unknownKeys: "strip" })
   );

   export class Plugin extends BSBObservable<InstanceType<typeof Config>> {
     static Config = Config;

     constructor(cfg: BSBObservableConstructor<InstanceType<typeof Config>>) {
       super(cfg);
     }

     debug(trace, pluginName, message, meta) {}
     info(trace, pluginName, message, meta) {}
     warn(trace, pluginName, message, meta) {}
     error(trace, pluginName, message: string | BSBError<any>, meta?) {}
     // metrics/tracing methods are optional; implement as needed
   }
   ```
3. Update README.md and docs/ (registry docs).
4. Ensure bsb-plugin.json exists (generated by bsb-plugin-cli build; do not edit manually).
5. Add bsb-tests.json entry when you need test config (optional, not user-facing).

Testing:
- Core suite from @bsb/tests runs for observable-* plugins.
- Add custom tests in tests/<plugin-id>/ if needed.

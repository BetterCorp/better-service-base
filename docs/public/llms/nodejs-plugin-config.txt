Config Plugin Guidance (Node.js)

Goal:
- Create a config-* plugin that loads configuration (file/env/remote).

Steps:
1. Create plugin folder: src/plugins/config-<name>.
2. Create index.ts with Config + Plugin (BSBConfig).
   - Config defines plugin metadata and config schema (only include fields you actually have).
   - Plugin implements getters for events/observable/service configs and reads config source.
   Example index.ts:
   ```ts
   import { BSBConfig, BSBConfigConstructor, createConfigSchema, Observable, PluginTypes, BSBError } from "@bsb/base";
   import { z } from "zod";
   import { parse } from "yaml";
   import * as fs from "node:fs";
   import * as path from "node:path";

   const Config = createConfigSchema(
     {
       name: "config-example",
       description: "Example config plugin",
       tags: ["config", "example"],
       documentation: ["./docs/config-example.md"]
     },
     z.object({
       BSB_PROFILE: z.string().optional().default("default"),
       BSB_CONFIG_FILE: z.string().optional().default("./sec-config.yaml")
     })
   );

   export class Plugin extends BSBConfig<InstanceType<typeof Config>> {
     static Config = Config;
     private _appConfig: any;
     private _profile: string = "default";

     constructor(cfg: BSBConfigConstructor<InstanceType<typeof Config>>) {
       super(cfg);
     }

     init(obs: Observable) {
       this._profile = this.config.BSB_PROFILE;
       const filePath = path.join(this.cwd, this.config.BSB_CONFIG_FILE);
       if (!fs.existsSync(filePath)) {
         throw new BSBError(obs.trace, "Cannot find config file at {filepath}", { filepath: filePath });
       }
       this._appConfig = parse(fs.readFileSync(filePath, "utf8").toString());
     }
     async getObservablePlugins() { return {}; }
     async getEventsPlugins() { return {}; }
     async getServicePlugins() { return {}; }
     async getPluginConfig(obs: Observable, pluginType: PluginTypes, plugin: string) { return null; }
     async getPlugins() { return []; }
   }
   ```
3. Update README.md and docs/ (registry docs).
4. Ensure bsb-plugin.json exists (generated by bsb-plugin-cli build; do not edit manually).
5. Add bsb-tests.json entry when you need test config (optional, not user-facing).

Testing:
- Config plugins are validated via @bsb/tests (config-default suite) and custom tests.
- Add custom tests in tests/<plugin-id>/ if needed.

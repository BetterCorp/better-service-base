Create A New Plugin

Goal:
- Add a new plugin in src/plugins/<plugin-name>.
- Node.js plugins are ESM-only. Use `"type": "module"`, `NodeNext`, and `.js` specifiers for relative imports.

Steps:
1. Pick a prefix: events-, observable-, config-, or service-.
2. Create plugin folder in src/plugins and add index.ts with Plugin + Config (see specific plugin doc).
3. Define Config metadata (name, description, tags, documentation).
4. Add README.md and docs/ for registry.
5. Ensure bsb-plugin.json exists (generated by bsb-plugin-cli build; do not edit manually).
6. Add bsb-tests.json entry when you need test config (optional, not user-facing).

Plugin types (Node.js):
- service-*: long-lived runtime services. Use events + observability. Lifecycle: init(obs), run(obs), dispose().
- events-*: event bus transport (emit/on/broadcast/streams).
- observable-*: logging/metrics/tracing backends.
- config-*: load configuration (e.g., file/env/remote vault). This plugin supplies config to the platform.

- Use @bsb/tests core suites.
- Add custom tests under tests/<plugin-id>/ if needed.

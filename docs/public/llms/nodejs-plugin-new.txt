Create A New Plugin

Goal:
- Add a new plugin in src/plugins/<plugin-name>.
- Node.js plugins are ESM-only. Use `"type": "module"`, `NodeNext`, and `.js` specifiers for relative imports.

Steps:
1. Pick a prefix: events-, observable-, config-, or service-.
2. Create plugin folder in src/plugins and add index.ts with Plugin + Config (see specific plugin doc).
3. Define Config metadata (name, description, tags, documentation).
4. Add README.md and docs/ for registry.
5. Ensure bsb-plugin.json exists (generated by bsb-plugin-cli build; do not edit manually).
6. Add bsb-tests.json entry when you need test config (optional, not user-facing).

Build/runtime rules:
- package.json build must be `"bsb-plugin-cli build"`, not `tsc`.
- Use BSB hooks for codegen/setup. Do not hide pre/post work in raw npm scripts around `tsc`.
- Do not run plugin runtime files directly with `node`, `tsx`, or `ts-node`; use `bsb-plugin-cli dev`, `bsb-plugin-cli start`, tests, or hooks so the BSB lifecycle is active.
- Keep each plugin self-contained. Store schemas, assets, helpers, generated inputs, and local docs in the plugin project/folder; reference them from other code when needed. Avoid shared side directories that are required for a plugin to build or run.

Event contract rules:
- An event name belongs to the plugin that owns/handles that contract.
- Do not define the same event name in two plugins. If plugin B needs to call plugin A, plugin B imports plugin A's generated client from `../../.bsb/clients/<plugin-id>.js` and calls the generated method.
- Generated/reference clients are flipped from the source plugin schema: source `onEvents`/`onReturnableEvents` become client emit methods; source `emitEvents`/`emitReturnableEvents` become client listener methods.
- For server-targeted delivery, use generated `*Specific(serverId, ...)` client methods instead of raw event internals.
- Example: service A has `onReturnableEvents["thing.get"]`; service B uses `new ServiceAClient(this).thingGet(...)`, not `emitReturnableEvents["thing.get"]`.

Plugin types (Node.js):
- service-*: long-lived runtime services. Use events + observability. Lifecycle: init(obs), run(obs), dispose().
- events-*: event bus transport (emit/on/broadcast/streams).
- observable-*: logging/metrics/tracing backends.
- config-*: load configuration (e.g., file/env/remote vault). This plugin supplies config to the platform.

Service lifecycle placement:
- constructor: class fields, generated clients, pure helpers. No network/database/listeners.
- init(obs): setup resources and register event handlers.
- run(obs): start long-lived external work after init: HTTP server listen, queue consume, scheduler/background loop.
- dispose(): close/stop what init/run opened.

- Use @bsb/tests core suites.
- Add custom tests under tests/<plugin-id>/ if needed.

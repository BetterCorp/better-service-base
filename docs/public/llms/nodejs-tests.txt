Testing (BSB Plugins)

Runner:
- Use npx @bsb/tests (or bsb-plugin-cli test).
- Auto-discovers plugins from bsb-plugin.json in the project root.

bsb-tests.json:
- Place next to bsb-plugin.json.
- Same structure as bsb-plugin.json, keyed by platform.
- Each plugin entry supports:
  - skip: true|false
  - default: { config, setup, dispose }
  - tests: [ { name, skip, config, setup, dispose } ]
Notes:
- bsb-plugin.json is generated by bsb-plugin-cli build; do not edit manually.
- bsb-tests.json is optional and used only for test config (not user-facing).

Setup/Dispose:
- setup can be a script path or { beforeAll, afterAll }.
- dispose is optional cleanup, run after tests.
- Scripts can be .js/.ts or any executable (bin).
- Paths are relative to the plugin root.

Custom tests:
- Place under tests/{plugin-id}/.
- Each file (not starting with "_") exports a single test function.
- _before/_after can be used per folder.
- If _before returns data, it is passed to each test in that folder.

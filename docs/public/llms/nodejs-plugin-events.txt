Events Plugin Guidance (Node.js)

Goal:
- Create an events-* plugin that transports events between services.
- Node.js plugins are ESM-only. Use `"type": "module"`, `NodeNext`, and `.js` specifiers for relative imports.

Steps:
1. Create plugin folder: src/plugins/events-<name>.
2. Create index.ts with Config + Plugin (BSBEvents).
   - Config defines plugin metadata and optional config schema (only include fields you actually have).
   - Do not pass `{ unknownKeys: "strip" }` to plugin config `av.object(...)`; BSB enforces `schema.parse(config, { unknownKeys: "strip" })` centrally at startup.
   - Plugin composes transport helpers (see events-default) or your own implementation.
   Example index.ts:
   ```ts
   import { BSBEvents, BSBEventsConstructor, createConfigSchema, Observable } from "@bsb/base";
   import * as av from "anyvali";
   import { Readable } from "node:stream";
   import {
     emit,
     broadcast,
     emitAndReturn,
     emitStreamAndReceiveStream
   } from "./events.js"; // your transport implementation

   const Config = createConfigSchema(
     {
       name: "events-example",
       description: "Example events plugin",
       tags: ["events", "example"],
       documentation: ["./docs/events-example.md"]
     },
     av.object({}) // optional config schema
   );

   export class Plugin extends BSBEvents<InstanceType<typeof Config>> {
     static Config = Config;

     private _emit: emit;
     private _broadcast: broadcast;
     private _ear: emitAndReturn;
     private _eas: emitStreamAndReceiveStream;

     constructor(cfg: BSBEventsConstructor<InstanceType<typeof Config>>) {
       super(cfg);
       this._emit = new emit(this.__internalObservable);
       this._broadcast = new broadcast(this.__internalObservable);
       this._ear = new emitAndReturn(this.__internalObservable);
       this._eas = new emitStreamAndReceiveStream(this.__internalObservable);
     }

     async init(obs: Observable) {}

     async onEvent(obs: Observable, pluginName: string, event: string, listener: (obs: Observable, args: any[]) => Promise<void>) {
       await this._emit.onEvent(obs, pluginName, event, listener);
     }
     async emitEvent(obs: Observable, pluginName: string, event: string, args: any[]) {
       await this._emit.emitEvent(obs, pluginName, event, args);
     }
     async onBroadcast(obs: Observable, pluginName: string, event: string, listener: (obs: Observable, args: any[]) => Promise<void>) {
       await this._broadcast.onBroadcast(obs, pluginName, event, listener);
     }
     async emitBroadcast(obs: Observable, pluginName: string, event: string, args: any[]) {
       await this._broadcast.emitBroadcast(obs, pluginName, event, args);
     }
     async onReturnableEvent(obs: Observable, pluginName: string, event: string, listener: (obs: Observable, args: any[]) => Promise<any>) {
       await this._ear.onReturnableEvent(obs, pluginName, event, listener);
     }
     async emitEventAndReturn(obs: Observable, pluginName: string, event: string, timeoutSeconds: number, args: any[]) {
       return await this._ear.emitEventAndReturn(obs, pluginName, event, timeoutSeconds, args);
     }
     async receiveStream(obs: Observable, pluginName: string, event: string, listener: (obs: Observable, error: Error | null, stream: Readable) => Promise<void>, timeoutSeconds?: number) {
       return this._eas.receiveStream(obs, pluginName, event, listener, timeoutSeconds);
     }
     async sendStream(obs: Observable, pluginName: string, event: string, streamId: string, stream: Readable) {
       return this._eas.sendStream(obs, pluginName, event, streamId, stream);
     }
   }
   ```
3. Update README.md and docs/ (registry docs).
4. Ensure bsb-plugin.json exists (generated by bsb-plugin-cli build; do not edit manually).
5. Add bsb-tests.json entry when you need test config (optional, not user-facing).

Testing:
- Core suite from @bsb/tests runs for events-* plugins.
- Add custom tests in tests/<plugin-id>/ if needed.

Node.js Bootstrap (BSB)

Goal:
- Create a minimal BSB Node.js project with plugins in src/plugins/* and a working runtime.

Prereqs:
- Node >= 23, npm >= 11.
- This project uses @bsb/base at runtime (framework + plugin CLI).

Bootstrap steps:
1. Create a new npm project (npm init) and install `@bsb/base` as a dev and peer dependency.
2. Make plugins directory `src/plugins/`
3. Add `/tsconfig.json` with:
```json
{
  "compilerOptions": {
    "target": "ES2022", // (or newer)
    "module": "commonjs",
    "lib": ["ES2022"],
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "outDir": "./lib",
    "rootDir": "./src",
    // Strict type checking - NO `any` types allowed!
    // BSB is designed to be fully type-safe across languages
    "strict": true, // Includes noImplicitAny
    "noImplicitAny": true, // Explicit: error on implicit any
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "moduleResolution": "node",
    "resolveJsonModule": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```
4. Add a default sec-config.yaml:
Note: config plugins are not defined here as they load the config (this file is loaded by the config-default). If you use a remote config plugin (e.g., config-vault), sec-config.yaml may not be used.
```yaml
default: # this is the deployment profile
  observable:
    observable-default:
      plugin: observable-default
      enabled: true
      config: {}
  events:
    events-default:                    # Use events-default for local, or events-rabbitmq for distributed
      plugin: events-default
      enabled: true
  services: {} # or actual plugins list
    # {mapped plugin name}:
    #   plugin: {actual plugin name}
    #   # package: "{npm package}" # Uncomment if not running in CWD
    #   enabled: true
    #   config: {} # Or actual config

```
5. Add the following to package.json:
```json 
{
  "files": [
    "lib/**/*",
    "README.md",
    "bsb-plugin.json",
    "bsb-tests.json"
  ],
  "scripts": {
    "build": "bsb-plugin-cli build",
    "clean": "bsb-plugin-cli clean",
    "start": "bsb-plugin-cli start",
    "dev": "bsb-plugin-cli dev",
    "test": "bsb-plugin-cli test",
    "publish:client": "bsb client publish"
  },
  "bsb": {}
}
```
Make sure there is a license defined otherwise default to: `(AGPL-3.0-only OR Commercial)`

Notes:
- bsb-plugin.json is generated by bsb-plugin-cli build; do not edit manually.
- bsb-tests.json is optional and used only for test config (not user-facing).

Recommended defaults (for dev):
Config: config-default.
Observable: observable-default.
Events: events-default.

(change plugins for production deployment - get plugins from BSB Registry for more information - https://io.bsbcode.dev/llms.txt)

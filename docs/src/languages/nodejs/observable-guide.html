Observable Pattern Guide

Complete guide to using the Observable pattern for unified observability in BSB v9

What is Observable?

Observable is BSB's unified interface for observability, combining logging, metrics, and distributed tracing into a single, cohesive API. Instead of juggling separate trace objects and logging calls, Observable provides everything you need in one immutable context.

Key Benefits:

  • Single interface for all observability needs
  • Automatic trace context propagation
  • Immutable attributes that safely propagate through operations
  • W3C trace context compatible
  • Resource context automatically included (service name, version, region, etc.)

See also: Migration Guide from v8 to v9

Core Concepts

Immutability

Observable instances are immutable. Methods like setAttribute() and span() return new Observable instances with updated context.

Automatic Context

All logs and metrics automatically include trace IDs, span IDs, and custom attributes without manual passing.

Resource Context

Every Observable includes resource information (service name, version, instance ID, environment, region) from the plugin configuration.

W3C Compatible

Trace IDs and span IDs follow W3C trace context standards for interoperability with other tracing systems.

Observable Properties

Property Type Description
trace DTrace Underlying trace object (W3C compatible)
traceId string Unique identifier for the entire trace
spanId string Unique identifier for this specific span
resource ResourceContext Service name, version, instance ID, environment, region
attributes Record<string, string | number | boolean> Custom attributes attached to this observable
log object Logging methods (debug, info, warn, error)
metrics object Metrics methods (counter, gauge, histogram, timer)

Logging with Observable

Observable provides four logging levels, all with automatic trace context inclusion:

Debug Logging

obs.log.debug(message: string, ...meta: SmartLogMeta): void

Used for detailed debugging information. Only logged in development mode by default.

// Simple message
obs.log.debug("Processing started");

// With metadata
obs.log.debug("Processing item {id} of {total}", {
  id: item.id,
  total: items.length
});

// Metadata is automatically replaced in placeholders
obs.log.debug("User {userId} performed action {action}", {
  userId: user.id,
  action: "login"
});
// Output: "User abc-123 performed action login"

Info Logging

obs.log.info(message: string, ...meta: SmartLogMeta): void

Used for general informational messages about application flow.

// Service lifecycle
obs.log.info("Service initialized successfully");

// Business events
obs.log.info("Order {orderId} created for customer {customerId}", {
  orderId: order.id,
  customerId: order.customerId
});

// With multiple metadata objects
obs.log.info("Processing complete", {
  duration: timer.stop(),
  itemsProcessed: items.length
});

Warning Logging

obs.log.warn(message: string, ...meta: SmartLogMeta): void

Used for potentially harmful situations that should be reviewed.

// Resource warnings
obs.log.warn("Connection pool at {percent}% capacity", {
  percent: 85
});

// Validation warnings
obs.log.warn("Deprecated API endpoint used: {endpoint}", {
  endpoint: req.path
});

// Rate limiting
obs.log.warn("Rate limit approaching for user {userId}", {
  userId: user.id
});

Error Logging

obs.log.error(messageOrError: string | Error, ...meta: SmartLogMeta): void

Used for error conditions. Supports both error objects and string messages.

// Error object
try {
  await dangerousOperation();
} catch (error) {
  obs.log.error(error as Error);
}

// Error with context
try {
  await processOrder(order);
} catch (error) {
  obs.log.error(error as Error, {
    orderId: order.id,
    customerId: order.customerId
  });
}

// String error message
obs.log.error("Failed to connect to database: {reason}", {
  reason: "Connection timeout"
});
Automatic Trace Context: All log methods automatically include the trace ID and span ID in the log output, making it easy to correlate logs with distributed traces.

Metrics with Observable

Observable provides methods to create and use four types of metrics:

Counters

obs.metrics.counter(name, description, help, labels?): Counter

Monotonically increasing values. Use for counting events, requests, errors, etc.

// Create a counter
const requestCounter = obs.metrics.counter(
  "http_requests_total",
  "Total HTTP requests",
  "Count of all HTTP requests received by the service",
  ["method", "status"]
);

// Increment counter
requestCounter.increment(1, {
  method: "GET",
  status: "200"
});

// Counter without labels
const errorCounter = obs.metrics.counter(
  "errors_total",
  "Total errors",
  "Count of all errors encountered"
);
errorCounter.increment(1);

Gauges

obs.metrics.gauge(name, description, help, labels?): Gauge

Values that can go up and down. Use for measuring current state like active connections, queue size, etc.

// Create a gauge
const activeConnections = obs.metrics.gauge(
  "active_connections",
  "Active database connections",
  "Number of currently active database connections"
);

// Set gauge value
activeConnections.set(10);

// Increment/decrement
activeConnections.increment(1); // Add 1
activeConnections.decrement(1); // Subtract 1

// With labels
const queueSize = obs.metrics.gauge(
  "queue_size",
  "Queue size",
  "Current size of processing queues",
  ["queue_name"]
);

queueSize.set(42, { queue_name: "orders" });

Histograms

obs.metrics.histogram(name, description, help, boundaries?, labels?): Histogram

Samples observations and counts them in configurable buckets. Use for measuring distributions like request durations, response sizes, etc.

// Create a histogram with custom boundaries
const requestDuration = obs.metrics.histogram(
  "http_request_duration_ms",
  "HTTP request duration",
  "Distribution of HTTP request durations in milliseconds",
  [10, 50, 100, 500, 1000, 5000], // Boundaries in ms
  ["method", "route"]
);

// Record observation
requestDuration.record(145, {
  method: "POST",
  route: "/api/users"
});

// Use with timer
const timer = obs.metrics.timer();
await processRequest();
const duration = timer.stop();
requestDuration.record(duration, {
  method: "GET",
  route: "/api/orders"
});

Timers

obs.metrics.timer(): Timer

Convenience method for measuring elapsed time. Returns duration in milliseconds.

// Basic timer usage
const timer = obs.metrics.timer();
await someOperation();
const duration = timer.stop();
obs.log.info("Operation took {ms}ms", { ms: duration });

// Use with histogram
const processTimer = obs.metrics.timer();
await processItems(items);
const processDuration = processTimer.stop();

const durationHistogram = obs.metrics.histogram(
  "process_duration_ms",
  "Process duration",
  "Time taken to process items"
);
durationHistogram.record(processDuration);

// Multiple timers
const dbTimer = obs.metrics.timer();
const results = await database.query();
const dbDuration = dbTimer.stop();

const serializeTimer = obs.metrics.timer();
const json = JSON.stringify(results);
const serializeDuration = serializeTimer.stop();

obs.log.info("DB: {db}ms, Serialize: {serialize}ms", {
  db: dbDuration,
  serialize: serializeDuration
});
Performance Tip: Create metrics once (usually in init()) and reuse them. Creating metrics repeatedly can impact performance.

Distributed Tracing

Observable makes distributed tracing simple with automatic parent-child span relationships:

Creating Spans

obs.span(name: string, attributes?): Observable

Creates a child span with automatic parent-child relationship. Always call end() when done.

// Basic span
const span = obs.span("database-query");
try {
  const results = await db.query("SELECT * FROM users");
  span.end({ rows: results.length });
  return results;
} catch (error) {
  span.error(error as Error);
  span.end({ success: false });
  throw error;
}

// Span with initial attributes
const userSpan = obs.span("fetch-user", {
  userId: user.id,
  cached: false
});

const userData = await fetchUser(user.id);
userSpan.end({ found: true });

// Nested spans
const orderSpan = obs.span("process-order", {
  orderId: order.id
});

// Validation span (child of orderSpan)
const validationSpan = orderSpan.span("validate-order");
await validateOrder(order);
validationSpan.end({ valid: true });

// Payment span (child of orderSpan)
const paymentSpan = orderSpan.span("process-payment");
const payment = await processPayment(order);
paymentSpan.end({ transactionId: payment.id });

// End parent span
orderSpan.end({ status: "completed" });

Recording Errors

obs.error(error: Error, attributes?): void

Records an error to both logs and the active span. Automatically includes error details.

// Simple error recording
const span = obs.span("risky-operation");
try {
  await doSomethingRisky();
  span.end({ success: true });
} catch (error) {
  span.error(error as Error);
  span.end({ success: false });
  throw error;
}

// Error with additional context
const processSpan = obs.span("process-batch");
try {
  await processBatch(items);
  processSpan.end({ itemsProcessed: items.length });
} catch (error) {
  processSpan.error(error as Error, {
    itemsProcessed: processedCount,
    itemsFailed: items.length - processedCount,
    retryable: true
  });
  processSpan.end();
  throw error;
}

Ending Spans

obs.end(attributes?): void

Completes the span and optionally records final attributes. Only affects spans created with span().

// End with final attributes
const span = obs.span("data-processing");
const results = await processData(data);
span.end({
  itemsProcessed: results.length,
  duration: timer.stop(),
  status: "success"
});

// End without attributes
const simpleSpan = obs.span("quick-operation");
await doSomething();
simpleSpan.end();

// Conditional attributes
const conditionalSpan = obs.span("conditional-operation");
const success = await tryOperation();
conditionalSpan.end({
  success: success,
  retries: success ? 0 : 3
});
Best Practice: Always end spans in a finally block or ensure they're ended in both success and error paths to avoid incomplete traces.

Attribute Propagation

One of the most powerful features of Observable is immutable attribute propagation. Attributes you set are automatically included in all child operations.

Setting Single Attributes

obs.setAttribute(key: string, value: string | number | boolean): Observable

Creates a new Observable with an additional attribute. The original Observable is unchanged.

// Add request context
const withRequest = obs.setAttribute("request.id", req.id);
withRequest.log.info("Handling request");
// Log automatically includes request.id

// Chain attributes
const withUser = withRequest.setAttribute("user.id", user.id);
withUser.log.info("User authenticated");
// Log includes both request.id and user.id

// Create span with inherited attributes
const span = withUser.span("process-data");
// Span automatically has request.id and user.id attributes
span.end();

Setting Multiple Attributes

obs.setAttributes(attrs: Record<string, string | number | boolean>): Observable

More efficient way to add multiple attributes at once.

// Set multiple attributes efficiently
const withContext = obs.setAttributes({
  "request.id": req.id,
  "user.id": req.userId,
  "request.method": req.method,
  "request.path": req.path
});

withContext.log.info("Processing request");
// All 4 attributes are included in the log

// Complex context
const orderContext = obs.setAttributes({
  "order.id": order.id,
  "customer.id": order.customerId,
  "order.total": order.total,
  "order.itemCount": order.items.length
});

const processSpan = orderContext.span("process-order");
// Span inherits all order attributes automatically
export class OrderService extends BSBService<ConfigSchema, EventSchemas> {
  public async handleOrderCreated(obs: Observable, order: Order) {
    // Add order context
    const withOrder = obs.setAttributes({
      "order.id": order.id,
      "customer.id": order.customerId,
      "order.total": order.total
    });

    withOrder.log.info("Processing new order");
    // Log includes order.id, customer.id, order.total

    // Validation span inherits all attributes
    const validationSpan = withOrder.span("validate-order");
    try {
      await this.validateOrder(order);
      validationSpan.end({ valid: true });
    } catch (error) {
      validationSpan.error(error as Error);
      // Error log includes all order attributes + error details
      validationSpan.end({ valid: false });
      throw error;
    }

    // Payment span also inherits all attributes
    const paymentSpan = withOrder.span("process-payment");
    const payment = await this.processPayment(order);

    // Add payment-specific attributes
    const withPayment = paymentSpan.setAttribute(
      "payment.id",
      payment.transactionId
    );
    withPayment.log.info("Payment processed");
    // Log now includes order attributes + payment.id

    paymentSpan.end({ success: true });

    withOrder.log.info("Order completed successfully");
    // All operations maintain consistent context
  }
}
Immutability Note: setAttribute() and setAttributes() return NEW Observable instances. The original Observable is unchanged. This is by design for safety and predictability.

Common Patterns and Best Practices

Pattern 1: Request Handler with Full Observability

export class APIService extends BSBService<ConfigSchema, EventSchemas> {
  private requestCounter: Counter;
  private requestDuration: Histogram;

  public async init(obs: Observable): Promise<void> {
    // Create metrics once
    this.requestCounter = obs.metrics.counter(
      "api_requests_total",
      "Total API requests",
      "Count of all API requests",
      ["method", "status"]
    );

    this.requestDuration = obs.metrics.histogram(
      "api_request_duration_ms",
      "API request duration",
      "Distribution of API request durations",
      [10, 50, 100, 500, 1000, 5000],
      ["method", "route"]
    );

    await this.events.onEvent(
      'http.request',
      obs,
      async (requestObs, req) => {
        await this.handleRequest(requestObs, req);
      }
    );
  }

  private async handleRequest(obs: Observable, req: Request) {
    // Add request context
    const withContext = obs.setAttributes({
      "request.id": req.id,
      "request.method": req.method,
      "request.path": req.path,
      "user.id": req.userId
    });

    // Start timer
    const timer = withContext.metrics.timer();

    // Create request span
    const requestSpan = withContext.span("handle-request");

    try {
      requestSpan.log.info("Processing request");

      // Process request (span automatically propagates)
      const result = await this.processRequest(requestSpan, req);

      // Record success metrics
      const duration = timer.stop();
      this.requestCounter.increment(1, {
        method: req.method,
        status: "200"
      });
      this.requestDuration.record(duration, {
        method: req.method,
        route: req.path
      });

      requestSpan.end({
        status: "success",
        duration: duration
      });

      return result;

    } catch (error) {
      // Record error
      requestSpan.error(error as Error);

      // Record failure metrics
      const duration = timer.stop();
      this.requestCounter.increment(1, {
        method: req.method,
        status: "500"
      });

      requestSpan.end({
        status: "error",
        duration: duration
      });

      throw error;
    }
  }

  private async processRequest(obs: Observable, req: Request) {
    // Span inherits all request context automatically
    const processSpan = obs.span("validate-and-execute");

    // Validation
    const validationSpan = processSpan.span("validate");
    await this.validate(req);
    validationSpan.end({ valid: true });

    // Execution
    const execSpan = processSpan.span("execute");
    const result = await this.execute(req);
    execSpan.end({ resultCount: result.length });

    processSpan.end();
    return result;
  }
}

Pattern 2: Background Job Processing

export class JobProcessor extends BSBService<ConfigSchema, EventSchemas> {
  private jobCounter: Counter;
  private processingGauge: Gauge;

  public async init(obs: Observable): Promise<void> {
    this.jobCounter = obs.metrics.counter(
      "jobs_processed_total",
      "Total jobs processed",
      "Count of all processed jobs",
      ["type", "status"]
    );

    this.processingGauge = obs.metrics.gauge(
      "jobs_processing",
      "Currently processing jobs",
      "Number of jobs currently being processed"
    );

    await this.events.onEvent(
      'job.queued',
      obs,
      async (jobObs, job) => {
        await this.processJob(jobObs, job);
      }
    );
  }

  private async processJob(obs: Observable, job: Job) {
    // Add job context
    const withJob = obs.setAttributes({
      "job.id": job.id,
      "job.type": job.type,
      "job.priority": job.priority
    });

    // Track active jobs
    this.processingGauge.increment(1);

    const jobSpan = withJob.span("process-job");

    try {
      jobSpan.log.info("Starting job processing");

      // Process with retries
      await this.processWithRetry(jobSpan, job);

      // Success
      this.jobCounter.increment(1, {
        type: job.type,
        status: "success"
      });

      jobSpan.end({ status: "completed" });

      await this.events.emitEvent('job.completed', jobSpan, {
        jobId: job.id,
        result: "success"
      });

    } catch (error) {
      jobSpan.error(error as Error);

      this.jobCounter.increment(1, {
        type: job.type,
        status: "failed"
      });

      jobSpan.end({ status: "failed" });

      await this.events.emitEvent('job.failed', jobSpan, {
        jobId: job.id,
        error: (error as Error).message
      });

    } finally {
      this.processingGauge.decrement(1);
    }
  }

  private async processWithRetry(
    obs: Observable,
    job: Job,
    maxRetries: number = 3
  ) {
    for (let attempt = 1; attempt <= maxRetries; attempt++) {
      const attemptSpan = obs.span("job-attempt", {
        attempt: attempt,
        maxRetries: maxRetries
      });

      try {
        await this.executeJob(attemptSpan, job);
        attemptSpan.end({ success: true });
        return; // Success
      } catch (error) {
        attemptSpan.error(error as Error);
        attemptSpan.end({ success: false });

        if (attempt === maxRetries) {
          throw error; // Final attempt failed
        }

        // Wait before retry
        const backoff = Math.pow(2, attempt) * 1000;
        attemptSpan.log.warn("Retrying in {ms}ms", { ms: backoff });
        await new Promise(resolve => setTimeout(resolve, backoff));
      }
    }
  }
}

Pattern 3: Cross-Service Communication

export class OrderService extends BSBService<ConfigSchema, EventSchemas> {
  public async createOrder(obs: Observable, orderData: OrderData) {
    const withOrder = obs.setAttribute("order.id", orderData.id);

    const orderSpan = withOrder.span("create-order");

    try {
      // Step 1: Validate user
      orderSpan.log.info("Validating user");
      const userSpan = orderSpan.span("validate-user");

      const validation = await this.events.emitEventAndReturn(
        'user.validate',
        userSpan,
        { userId: orderData.userId },
        5
      );

      if (!validation.valid) {
        userSpan.end({ valid: false });
        throw new Error("Invalid user");
      }
      userSpan.end({ valid: true });

      // Step 2: Check inventory
      orderSpan.log.info("Checking inventory");
      const inventorySpan = orderSpan.span("check-inventory");

      const inventory = await this.events.emitEventAndReturn(
        'inventory.check',
        inventorySpan,
        { items: orderData.items },
        5
      );

      if (!inventory.available) {
        inventorySpan.end({ available: false });
        throw new Error("Insufficient inventory");
      }
      inventorySpan.end({ available: true });

      // Step 3: Process payment
      orderSpan.log.info("Processing payment");
      const paymentSpan = orderSpan.span("process-payment");

      const payment = await this.events.emitEventAndReturn(
        'payment.process',
        paymentSpan,
        {
          amount: orderData.total,
          customerId: orderData.userId
        },
        10
      );

      if (!payment.success) {
        paymentSpan.end({ success: false });
        throw new Error("Payment failed");
      }

      const withPayment = paymentSpan.setAttribute(
        "payment.id",
        payment.transactionId
      );
      withPayment.log.info("Payment successful");
      paymentSpan.end({ success: true });

      // Step 4: Create order
      orderSpan.log.info("Creating order record");
      const order = await this.saveOrder(orderSpan, orderData);

      // Emit order created event
      await this.events.emitEvent('order.created', orderSpan, {
        orderId: order.id,
        customerId: orderData.userId,
        total: orderData.total
      });

      orderSpan.end({
        status: "success",
        orderId: order.id
      });

      return order;

    } catch (error) {
      orderSpan.error(error as Error);
      orderSpan.end({ status: "failed" });
      throw error;
    }
  }
}

Performance Considerations

Performance Best Practices

  • Create metrics once: Create counters, gauges, and histograms in init(), not in request handlers
  • Reuse Observables: Don't create new attributes for every operation if not needed
  • Batch attributes: Use setAttributes() for multiple attributes instead of chaining setAttribute()
  • End spans properly: Always end spans to avoid memory leaks in tracing systems
  • Use appropriate log levels: Avoid excessive debug logging in production
// ❌ BAD: Creating metrics repeatedly
public async handleRequest(obs: Observable, req: Request) {
  const counter = obs.metrics.counter("requests", "...", "...");
  counter.increment(1); // Creates metric every request!
}

// ✅ GOOD: Create once, reuse
private requestCounter: Counter;

public async init(obs: Observable) {
  this.requestCounter = obs.metrics.counter("requests", "...", "...");
}

public async handleRequest(obs: Observable, req: Request) {
  this.requestCounter.increment(1); // Reuses existing metric
}

// ❌ BAD: Chaining setAttribute repeatedly
const obs1 = obs.setAttribute("a", 1);
const obs2 = obs1.setAttribute("b", 2);
const obs3 = obs2.setAttribute("c", 3);
const obs4 = obs3.setAttribute("d", 4);

// ✅ GOOD: Use setAttributes
const withContext = obs.setAttributes({
  a: 1,
  b: 2,
  c: 3,
  d: 4
});

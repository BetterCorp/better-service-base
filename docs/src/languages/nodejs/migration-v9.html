Migration Guide: BSB v9

Complete guide for migrating from DTrace to the unified Observable pattern

What's New in v9

BSB v9 introduces the Observable pattern - a unified interface for observability that combines logging, metrics, and distributed tracing into a single, cohesive API. This replaces the previous DTrace parameter with a more powerful and intuitive Observable context.

  • Unified API: One interface for logs, metrics, and traces
  • Automatic context: Trace IDs and span relationships are automatic
  • Immutable attributes: Safely propagate context through operations
  • Better DX: More intuitive and discoverable API

See also: Complete Observable Usage Guide

What Changed

DTrace → Observable

The most significant change in v9 is the replacement of the DTrace parameter with Observable across all plugin lifecycle methods and event handlers.

v8 (DTrace)

// v8: DTrace parameter
public async init(trace: DTrace): Promise<void> {
  this.log.info(trace, "Starting service");

  await this.events.onEvent(
    'user.created',
    trace,
    async (itrace, data) => {
      this.log.debug(itrace, "New user");
    }
  );
}

v9 (Observable)

// v9: Observable parameter
public async init(obs: Observable): Promise<void> {
  obs.log.info("Starting service");

  await this.events.onEvent(
    'user.created',
    obs,
    async (childObs, data) => {
      childObs.log.debug("New user");
    }
  );
}

Logging Changes

Logging is now accessed directly through the Observable context, eliminating the need to pass trace objects separately.

v8

// v8: Separate log calls with trace
this.log.debug(trace, "Processing item");
this.log.info(trace, "User: {email}", {
  email: user.email
});
this.log.warn(trace, "Rate limit hit");
this.log.error(trace, error);

v9

// v9: Log through Observable
obs.log.debug("Processing item");
obs.log.info("User: {email}", {
  email: user.email
});
obs.log.warn("Rate limit hit");
obs.log.error(error);

Metrics Changes

Metrics are created through the Observable context, automatically including resource information.

v8

// v8: Create metrics via this.metrics
const counter = this.metrics.createCounter(
  trace,
  "requests_total",
  "Total requests",
  "Count of all incoming requests",
  ["method", "status"]
);

counter.increment(trace, 1, {
  method: "GET",
  status: "200"
});

v9

// v9: Create metrics via Observable
const counter = obs.metrics.counter(
  "requests_total",
  "Total requests",
  "Count of all incoming requests",
  ["method", "status"]
);

counter.increment(1, {
  method: "GET",
  status: "200"
});

Distributed Tracing

Creating child spans and propagating context is now much simpler with Observable.

v8

// v8: Manual span creation
const childTrace = this.metrics.createTrace(
  trace,
  "database-query"
);

try {
  await this.queryDatabase();
  this.metrics.endTrace(childTrace);
} catch (error) {
  this.metrics.errorTrace(childTrace, error);
  this.metrics.endTrace(childTrace);
  throw error;
}

v9

// v9: Automatic span management
const span = obs.span("database-query");

try {
  await this.queryDatabase();
  span.end({ status: "success" });
} catch (error) {
  span.error(error);
  span.end({ status: "failed" });
  throw error;
}

Breaking Changes

  • Method signatures changed: All init() and run() methods now take Observable instead of DTrace
  • Event handlers changed: All event handlers receive Observable as first parameter
  • Logging API changed: Use obs.log.method() instead of this.log.method(trace, ...)
  • Metrics API changed: Create metrics via obs.metrics instead of this.metrics
  • Tracing API changed: Use obs.span(), obs.error(), obs.end() instead of separate trace methods
  • No backward compatibility: DTrace is no longer supported in plugin APIs (internal use only)

Step-by-Step Migration

  1. Update Plugin Lifecycle Methods

    Replace DTrace with Observable in init() and run() methods:

export class Plugin extends BSBService<ConfigSchema, EventSchemas> {
  public async init(trace: DTrace): Promise<void> {
    this.log.info(trace, "Initializing plugin");
  }

  public async run(trace: DTrace): Promise<void> {
    this.log.info(trace, "Plugin running");
  }
}
    import { BSBService, BSBServiceConstructor, Observable } from '@bettercorp/service-base';

export class Plugin extends BSBService<ConfigSchema, EventSchemas> {
  public async init(obs: Observable): Promise<void> {
    obs.log.info("Initializing plugin");
  }

  public async run(obs: Observable): Promise<void> {
    obs.log.info("Plugin running");
  }
}

    Create metrics through Observable instead of this.metrics:

    public async init(trace: DTrace): Promise<void> {

  5. Update Distributed Tracing

    Use Observable's span API for creating child spans:

    public async processOrder(trace: DTrace, order: Order) {
  const childTrace = this.metrics.createTrace(
    trace,
    "process-order",
    { orderId: order.id }
  );

  try {
    // Validation span
    const validationTrace = this.metrics.createTrace(
      childTrace,
      "validate-order"
    );
    await this.validateOrder(order);
    this.metrics.endTrace(validationTrace);

    // Processing span
    const processingTrace = this.metrics.createTrace(
      childTrace,
      "enrich-order"
    );
    const enriched = await this.enrichOrder(order);
    this.metrics.endTrace(processingTrace);

    this.metrics.endTrace(childTrace, { status: "success" });
  } catch (error) {
    this.metrics.errorTrace(childTrace, error);
    this.metrics.endTrace(childTrace, { status: "failed" });
    throw error;
  }
}
    public async processOrder(obs: Observable, order: Order) {
  const orderSpan = obs.span("process-order", {
    orderId: order.id
  });

  try {
    // Validation span
    const validationSpan = orderSpan.span("validate-order");
    await this.validateOrder(order);
    validationSpan.end();

    // Processing span
    const processingSpan = orderSpan.span("enrich-order");
    const enriched = await this.enrichOrder(order);
    processingSpan.end();

    orderSpan.end({ status: "success" });
  } catch (error) {
    orderSpan.error(error);
    orderSpan.end({ status: "failed" });
    throw error;
  }
}

  6. Use Attribute Propagation

    One of the most powerful features of Observable is immutable attribute propagation:

    public async handleRequest(obs: Observable, req: Request) {
  // Add request context
  const withRequest = obs.setAttribute("request.id", req.id);
  withRequest.log.info("Handling request");

  // Add user context
  const withUser = withRequest.setAttribute("user.id", req.userId);

  // All operations inherit attributes
  const span = withUser.span("process-request");
  // span automatically has request.id and user.id attributes

  span.log.info("Processing");
  // This log automatically includes request.id and user.id

  span.end();
}
    Pro Tip: Use setAttributes() to add multiple attributes at once for better performance.

Common Migration Patterns

Pattern 1: Service with Event Handlers

// v8
export class Plugin extends BSBService<ConfigSchema, EventSchemas> {
  private requestCounter: Counter;

  public async init(trace: DTrace): Promise<void> {
    this.log.info(trace, "Initializing order service");

    this.requestCounter = this.metrics.createCounter(
      trace,
      "orders_total",
      "Total orders",
      "Total number of orders processed",
      ["status"]
    );

    await this.events.onEvent(
      'order.created',
      trace,
      async (itrace, order) => {
        this.log.info(itrace, "Processing order {orderId}", {
          orderId: order.id
        });

        try {
          await this.processOrder(itrace, order);
          this.requestCounter.increment(itrace, 1, { status: "success" });
        } catch (error) {
          this.log.error(itrace, error);
          this.requestCounter.increment(itrace, 1, { status: "failed" });
        }
      }
    );
  }

  private async processOrder(trace: DTrace, order: Order) {
    const childTrace = this.metrics.createTrace(trace, "process-order");
    // ... processing logic
    this.metrics.endTrace(childTrace);
  }
}

    const span = obs.span("process-order");
    // ... processing logic
    span.end();
  }
}

Pattern 2: Timed Operations with Metrics

// v8
public async handleRequest(trace: DTrace, data: any) {
  const timer = this.metrics.createTimer(trace);

  try {
    await this.processRequest(trace, data);
    const duration = timer.stop(trace);
    this.log.info(trace, "Request took {ms}ms", { ms: duration });
  } catch (error) {
    this.log.error(trace, error);
    throw error;
  }
}

// v9
public async handleRequest(obs: Observable, data: any) {
  const timer = obs.metrics.timer();

  try {
    await this.processRequest(obs, data);
    const duration = timer.stop();
    obs.log.info("Request took {ms}ms", { ms: duration });
  } catch (error) {
    obs.log.error(error);
    throw error;
  }
}

Pattern 3: Cross-Service Communication

// v8
public async validateAndCreate(trace: DTrace, user: UserData) {
  // Call validation service
  const validation = await this.events.emitEventAndReturn(
    'user.validate',
    trace,
    { email: user.email, password: user.password },
    5
  );

  if (!validation.valid) {
    this.log.warn(trace, "Validation failed: {errors}", {
      errors: validation.errors
    });
    throw new Error("Invalid user data");
  }

  // Create user
  await this.events.emitEvent('user.created', trace, user);
  this.log.info(trace, "User created successfully");
}

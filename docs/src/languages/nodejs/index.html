Node.js Implementation

Complete guide to building BSB plugins with Node.js and typed event interfaces

📋 Requirements

  • Node.js ≥ 23.0.0
  • npm ≥ 11.0.0
  • Docker (runtime - production)
  • TypeScript & Node.js (optional - for plugin development)

⚡ Quick Installation (Dev)

For local plugin development and type support:

npm install @bsb/base

# Create your plugin structure (dev)
mkdir -p src/plugins/service-my-api
echo 'console.log("BSB service ready!");' > src/plugins/service-my-api/index.ts

🎯 How BSB Works

BSB is designed to run as a standalone Docker container that loads and runs plugins (published separately). In production you deploy the container and provide plugins via BSB_PLUGIN_DIR or BSB_PLUGINS. Installing BSB as a dependency is only for contributors and local plugin development.

📖 BSB v9 Documentation

BSB v9 introduces the Observable pattern for unified observability. Check out our comprehensive guides:

  • Migration Guide: v8 to v9 - Step-by-step migration from DTrace to Observable
  • Observable Pattern Guide - Complete usage guide with examples
  • API Reference (TypeDoc) - Full TypeScript API documentation

🔎 Discovery & Runtime

  • Production: run the BSB container and provide plugins via BSB_PLUGIN_DIR
  • Optional: specify explicit versions with BSB_PLUGINS (e.g. service-my-api@1.2.3;observable-default@latest)
  • Development: install BSB in your plugin project for types and run locally if needed

🏗️ Project Structure

Container-first plugin repository layout (recommended for production):

/mnt/plugins/ @org/service-my-api/ latest/ package.json lib/plugins/service-my-api/index.js 1.2.3/ package.json lib/plugins/service-my-api/index.js @org/logging-custom/ latest/ package.json lib/plugins/logging-custom/index.js @org/metrics-prometheus/ latest/ package.json lib/plugins/metrics-prometheus/index.js

Plugin development project structure (local dev):

my-plugin/ ├── package.json ├── src/ │ └── plugins/ │ └── service-my-api/ │ └── index.ts └── lib/ (built) └── plugins/service-my-api/index.js

🐳 Container Quick Start

Run the official BSB container and mount your plugin repository:

docker run --rm \
  -v $(pwd)/plugins:/mnt/plugins:ro \
  -e BSB_PLUGIN_DIR=/mnt/plugins \
  betterweb/service-base:node
Note: This is the supported production deployment.

  3. Create Your First Service Plugin

    Implement a basic service plugin with typed events and the correct lifecycle (constructor → init → run → dispose):

    import { BSBService, BSBServiceConstructor, BSBServiceClientDefinition, Observable, z } from '@bsb/base';
import { createFireAndForgetEvent, createReturnableEvent, createBroadcastEvent } from '@bsb/base';

// Schema-first event definitions with automatic validation
export const EventSchemas = {
  // Events this service emits (fire-and-forget)
  emitEvents: {
    'notification.send': createFireAndForgetEvent(
      z.object({
        type: z.enum(['welcome', 'alert']),
        userId: z.string().uuid(),
        message: z.string().optional()
      }),
      'Send notification to user'
    )
  },

  // Events this service listens to (fire-and-forget)
  onEvents: {
    'user.created': createFireAndForgetEvent(
      z.object({
        userId: z.string().uuid(),
        email: z.string().email()
      }),
      'Handle user creation event'
    )
  },

  // Returnable events this service emits (RPC-like requests from this service)
  emitReturnableEvents: {
    'data.fetch': createReturnableEvent(
      z.object({
        userId: z.string().uuid()
      }),
      z.object({
        userData: z.record(z.unknown())
      }),
      'Fetch user data from external service'
    )
  },

  // Returnable events this service listens to (RPC-like requests to this service)
  onReturnableEvents: {
    'api.getUser': createReturnableEvent(
      z.object({
        userId: z.string().uuid()
      }),
      z.object({
        userId: z.string().uuid(),
        status: z.string()
      }),
      'Get user status information'
    )
  },

  // Broadcast events this service emits (all listeners receive)
  emitBroadcast: {
    'system.alert': createBroadcastEvent(
      z.object({
        level: z.enum(['info', 'warning', 'error']),
        message: z.string(),
        timestamp: z.string().datetime()
      }),
      'System-wide alert broadcast'
    )
  },

  // Broadcast events this service listens to
  onBroadcast: {
    'system.maintenance': createBroadcastEvent(
      z.object({
        startTime: z.string().datetime(),
        duration: z.number().min(0)
      }),
      'Listen for maintenance mode broadcasts'
    )
  }
} as const; // 🔑 Critical for type safety!

// Learn more about the Observable pattern:
// - Migration Guide: /languages/nodejs/migration-v9.html
// - Observable Guide: /languages/nodejs/observable-guide.html

// IMPORTANT: Export class name MUST be `Plugin`
export class Plugin extends BSBService {
  public static PLUGIN_CLIENT: BSBServiceClientDefinition = {
    name: "service-my-api",
  }
  public initBeforePlugins?: string[] | undefined;
  public initAfterPlugins?: string[] | undefined;
  public runBeforePlugins?: string[] | undefined;
  public runAfterPlugins?: string[] | undefined;

  constructor(config: BSBServiceConstructor) {
    super({
      ...config,
      eventSchemas: EventSchemas
    });
  }

  public async init(obs: Observable): Promise {
    // Listen for user.created events (with automatic validation)
    await this.events.onEvent('user.created', obs, async (childObs, data) => {
      // Emit a notification when a user is created (with automatic validation)
      await this.events.emitEvent('notification.send', childObs, {
        type: 'welcome',
        userId: data.userId,
        message: `Welcome ${data.email}!`
      });
    });

    // Expose a returnable (RPC-like) handler (with automatic validation)
    await this.events.onReturnableEvent('api.getUser', obs, async (childObs, data) => {
      // Typically fetch from a DB; returning inline for example
      return { userId: data.userId, status: 'active' }; // Return data is validated automatically
    });
  }

  public async run(obs: Observable): Promise {
    // Service is ready - emit a system alert as an example
    await this.events.emitBroadcast('system.alert', obs, {
      level: 'info',
      message: 'My API service is now running',
      timestamp: new Date().toISOString()
    });
  }
}

  5. Create Plugin Config (Optional)

    Define configuration schema and metadata for your plugin:

    import { BSBPluginConfig, z } from '@bsb/base';

// Define your config schema using Zod
const MyConfigSchema = z.object({
  maxUsers: z.number().default(1000),
  features: z.object({
    notifications: z.boolean().default(true),
    analytics: z.boolean().default(false)
  }),
  apiKeys: z.object({
    external: z.string().optional()
  }).optional()
});

export class Config extends BSBPluginConfig<typeof MyConfigSchema> {
  public validationSchema = MyConfigSchema;

  // Optional: Config migration for version upgrades
  public async migrate(
    versionFrom: string | null,
    versionTo: string,
    existingConfig?: z.infer<typeof MyConfigSchema>
  ) {
    // First-time setup
    if (versionFrom === null) {
      return {
        maxUsers: 1000,
        features: {
          notifications: true,
          analytics: false
        }
      };
    }
    
    // Handle version migrations
    if (versionFrom === '1.0.0' && versionTo === '1.1.0') {
      return {
        ...existingConfig,
        features: {
          ...existingConfig?.features,
          analytics: false // New feature added in 1.1.0
        }
      };
    }

    return existingConfig || {};
  }
}

🔄 Event Communication

BSB services communicate exclusively through typed events:

// Fire-and-forget events (single object parameter)
await this.events.emitEvent('notification.send', obs, {
  type: 'welcome',
  userId: 'user-123',
  message: 'Welcome!'
});

// Returnable events (RPC-style, single object parameter)
const result = await this.events.emitEventAndReturn('api.getUser', obs, {
  userId: 'user-123'
}, 5); // timeout in seconds

// Broadcast events (all listeners receive, single object parameter)
await this.events.emitBroadcast('system.alert', obs, {
  level: 'info',
  message: 'System status update',
  timestamp: new Date().toISOString()
});

// Listen to events (object parameter in handler)
await this.events.onEvent('user.created', obs, async (childObs, data) => {
  console.log(`New user: ${data.email}`);
});

// Expose returnable methods (object parameter and return)
await this.events.onReturnableEvent('api.getUser', obs, async (childObs, data) => {
  return { userId: data.userId, status: 'active' };
});

// Listen to broadcast events
await this.events.onBroadcast('system.maintenance', obs, async (childObs, data) => {
  console.log(`Maintenance starts at: ${data.startTime}`);
});

🔒 Event Validation & Performance

BSB supports optional Zod-based validation for enhanced runtime type safety:

⚖️ Performance Considerations

  • Validation overhead: ~1-5ms per event (depends on schema complexity)
  • Production toggle: Set BSB_VALIDATE_EVENTS=false to disable validation
  • Selective validation: Validate only critical events for optimal performance
  • Performance monitoring: Automatic logging of slow validations
# Enable/disable validation globally
BSB_VALIDATE_EVENTS=true

# Performance tuning
BSB_VALIDATION_PERFORMANCE_THRESHOLD=10  # Log validations slower than 10ms
BSB_VALIDATION_STRICT=true               # Throw errors (false = log warnings)

// Complex validation schemas with nested objects and arrays
export const AdvancedEventSchemas = {
  onEvents: {
    'order.process': createFireAndForgetEvent(
      z.object({
        id: z.string().uuid(),
        items: z.array(z.object({
          productId: z.string().uuid(),
          quantity: z.number().positive(),
          price: z.number().positive()
        })).min(1),
        customer: z.object({
          id: z.string().uuid(),
          email: z.string().email(),
          tier: z.enum(['basic', 'premium'])
        }),
        metadata: z.record(z.unknown()).optional()
      }),
      'Process customer order with complex validation'
    )
  },
  
  emitReturnableEvents: {
    'payment.process': createReturnableEvent(
      z.object({
        orderId: z.string().uuid(),
        amount: z.number().positive(),
        currency: z.string().length(3),
        paymentMethod: z.enum(['card', 'bank', 'crypto'])
      }),
      z.object({
        transactionId: z.string(),
        status: z.enum(['success', 'failed', 'pending']),
        fees: z.number().min(0),
        processedAt: z.string().datetime()
      }),
      'Process payment with detailed response'
    )
  }
} as const;

// Use in your plugin class
export class AdvancedPlugin extends BSBService {
  constructor(config: BSBServiceConstructor) {
    super({
      ...config,
      eventSchemas: AdvancedEventSchemas
    });
  }
}

🐳 Container Deployment

Deploy the BSB container and mount your prebuilt plugins:

docker run --rm \
  -v /path/to/plugins:/mnt/plugins:ro \
  -e BSB_PLUGIN_DIR=/mnt/plugins \
  betterweb/service-base:node
services:
  bsb:
    image: betterweb/service-base:node
    environment:
      - BSB_PLUGIN_DIR=/mnt/plugins
      - BSB_PROFILE=default
    volumes:
      - ./plugins:/mnt/plugins:ro
    ports:
      - "8080:8080"  # if your service exposes HTTP
Important: Services communicate through the event bus and can run together in the same container or separately across multiple containers. The event bus handles all communication routing automatically.

logging-default

Console logging with structured output and trace propagation

Overview

The logging-default plugin outputs logs to the console with color coding and structured formatting. All BSB implementations provide the same logging interface with four log levels.

Logging Interface

  • debug(trace, message, meta?) - Development debugging (gray)
  • info(trace, message, meta?) - General information (blue)
  • warn(trace, message, meta?) - Potential issues (yellow)
  • error(trace, message, error?, meta?) - Failures (red)

Log Levels

Level Use Case Production
debug Detailed debugging information Usually disabled
info General operational messages Enabled
warn Potential issues, degraded behavior Enabled
error Failures requiring attention Enabled

Usage

Access the logger via this.log in your service:

import { BSBService, BSBServiceConstructor } from "@bettercorp/service-base";

export class Plugin extends BSBService {
  constructor(context: BSBServiceConstructor) {
    super(context);
  }

  async init(): Promise<void> {
    // Debug - development only
    this.log.debug("Processing item {itemId}", { itemId: "123" });

    // Info - general information
    this.log.info("User {userId} logged in", { userId: "user-456" });

    // Warn - potential issues
    this.log.warn("Rate limit approaching for {ip}", { ip: "192.168.1.1" });

    // Error - failures (with optional Error object)
    this.log.error("Failed to process order {orderId}", new Error("DB timeout"), {
      orderId: "789"
    });
  }
}

Structured Logging

Use template placeholders with metadata objects:

// The message template uses {placeholders}
// The metadata object provides the values
this.log.info("Order {orderId} placed by {userId} for {amount}", {
  orderId: "ORD-123",
  userId: "USR-456",
  amount: "$99.99"
});

// Output:
// [INFO] [service-orders] Order ORD-123 placed by USR-456 for $99.99

Trace Propagation

Traces connect logs across async operations:

async init(): Promise<void> {
  this.log.info("Initializing service");

  // Event handlers receive traces that connect to the original request
  await this.events.onReturnableEvent(
    "order.create",
    async (data) => {
      // This log is connected to the request that triggered the event
      this.log.info("Creating order for {userId}", { userId: data.userId });
      return { orderId: "new-123" };
    }
  );
}

When to Replace

Replace logging-default when you need:

  • Centralized log aggregation (Graylog, ELK, Splunk)
  • Log shipping to cloud services (Datadog, CloudWatch)
  • JSON-formatted output for log parsing
  • Log retention and search capabilities

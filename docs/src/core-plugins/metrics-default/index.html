metrics-default

Basic metrics collection with counters, gauges, and histograms

Overview

The metrics-default plugin provides basic metrics primitives for tracking application performance. All BSB implementations support the same metric types and tracing interface.

Metric Types

Counter

Cumulative values that only increase (requests, errors)

Gauge

Values that go up or down (queue size, connections)

Histogram

Value distributions (latency, payload size)

Trace/Span

Distributed tracing across services

Metrics Interface

  • createCounter(name, description, unit, labels) - Create a counter
  • createGauge(name, description, unit, labels) - Create a gauge
  • createHistogram(name, description, unit, labels) - Create a histogram
  • createTrace(name, attributes) - Create a trace
  • createSpan(trace, name) - Create a span within a trace

Usage

Counters

Track cumulative values that only increase:

// Create a counter
const requestCounter = this.metrics.createCounter(
  "requests_total",
  "Total requests processed",
  "requests",
  ["service-api"]
);

// Increment by 1
requestCounter.increment();

// Increment by specific amount
requestCounter.increment(5);

Gauges

Track values that can go up or down:

// Create a gauge
const queueGauge = this.metrics.createGauge(
  "queue_size",
  "Current queue size",
  "items",
  ["service-worker"]
);

// Set absolute value
queueGauge.set(42);

// Increment/decrement
queueGauge.increment();
queueGauge.decrement(5);

Histograms

Track value distributions:

// Create a histogram
const durationHistogram = this.metrics.createHistogram(
  "request_duration_ms",
  "Request duration in milliseconds",
  "ms",
  ["service-api"]
);

// Record observations
durationHistogram.observe(45.2);
durationHistogram.observe(123.5);

// Common pattern: measure operation duration
const start = Date.now();
await performOperation();
durationHistogram.observe(Date.now() - start);

Distributed Tracing

Create traces and spans for distributed tracing:

// Create a trace for an operation
const trace = this.metrics.createTrace("processOrder", {
  orderId: "123",
  userId: "user-456"
});

// Create spans for sub-operations
const dbSpan = this.metrics.createSpan(trace.trace, "database_query");
await queryDatabase();
dbSpan.end();

const apiSpan = this.metrics.createSpan(trace.trace, "external_api_call");
await callExternalAPI();
apiSpan.end();

// End the trace when done
trace.end();

Integration Example

Combining metrics in a real service:

export class Plugin extends BSBService {
  private requestCounter!: Counter;
  private errorCounter!: Counter;
  private latencyHistogram!: Histogram;
  private activeConnections!: Gauge;

  async init(): Promise<void> {
    // Initialize metrics
    this.requestCounter = this.metrics.createCounter(
      "http_requests_total", "Total HTTP requests", "requests", []
    );
    this.errorCounter = this.metrics.createCounter(
      "http_errors_total", "Total HTTP errors", "errors", []
    );
    this.latencyHistogram = this.metrics.createHistogram(
      "http_request_duration_ms", "Request latency", "ms", []
    );
    this.activeConnections = this.metrics.createGauge(
      "active_connections", "Active connections", "connections", []
    );
  }

  async handleRequest(req: Request): Promise<Response> {
    const start = Date.now();
    this.requestCounter.increment();
    this.activeConnections.increment();

    try {
      const response = await processRequest(req);
      return response;
    } catch (error) {
      this.errorCounter.increment();
      throw error;
    } finally {
      this.activeConnections.decrement();
      this.latencyHistogram.observe(Date.now() - start);
    }
  }
}

When to Replace

Replace metrics-default when you need:

  • Prometheus scraping endpoint
  • OpenTelemetry export
  • Monitoring dashboards (Grafana, Datadog)
  • Long-term metrics storage and querying

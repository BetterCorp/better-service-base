Build Hooks

Run custom npm scripts at specific points in the BSB build and dev pipelines. Integrate code generation, transforms, and external tooling into your build process.

Overview

Build hooks let you inject your own scripts into the BSB build pipeline. They run at well-defined points between BSB's internal steps, so you can generate code from schemas, run transforms before compilation, or set up your dev environment.

Hooks are configured in package.json under the bsb.hooks key. Values reference npm script names from your scripts section -- no raw shell commands.

Configuration

Add a hooks object to your bsb config in package.json. Each hook accepts either a single script name (string) or an array of script names that run sequentially.

{
  "name": "@myorg/my-plugin",
  "scripts": {
    "generate-api-types": "openapi-typescript ./openapi.yaml -o src/plugins/service-api/generated/openapi.ts",
    "post-bundle": "vite build --config worker.vite.config.ts",
    "lint-output": "eslint lib",
    "seed-db": "prisma db seed"
  },
  "bsb": {
    "hooks": {
      "afterSchemas": "generate-api-types",
      "afterCompile": ["post-bundle", "lint-output"],
      "beforeDev": "seed-db"
    }
  }
}

Safety by design

Hook values must be npm script names, not arbitrary commands. This keeps hooks auditable -- everything runs through your declared scripts section.

If a hook references a script name that does not exist in scripts, the build fails with a clear error message.

Available Hooks

Hook When Modes
beforeSchemas Before schema extraction and virtual client generation build + dev
afterSchemas After schema generation, before TypeScript compilation build + dev
beforeCompile Before TypeScript compilation starts build + dev
afterCompile After TypeScript compilation succeeds build + dev
afterBuild After full build completes (metadata, manifests, cache) build only
beforeDev Once before the dev server starts for the first time dev only

9
afterBuild

Your hook runs here -- build is fully complete

Dev Mode Behavior

During bsb-plugin-cli dev, hooks behave slightly differently:

  • beforeSchemas, afterSchemas, beforeCompile, and afterCompile run on both initial startup and file-change rebuilds
  • beforeDev runs only once, before the first service start -- not on rebuilds
  • afterBuild does not run in dev mode
  • Schema hooks are skipped for config-only or asset-only changes
  • Compile hooks are skipped when only non-TypeScript files change

Error handling

Hook failure behavior differs between build and dev modes:

  • Build mode: Hook failure aborts the build immediately. A failing codegen step should not produce a broken build.
  • Dev mode: Hook failure is non-fatal. The current service instance keeps running, and the error is logged. This matches how typecheck failures work in dev -- you fix the issue, save, and the next rebuild retries.

Common Use Cases

Custom code generation from schemas

The most common use case: generate types, API clients, or documentation from BSB's extracted schemas before TypeScript compilation.

{
  "scripts": {
    "generate-api-client": "openapi-typescript ./openapi.yaml -o src/plugins/service-api/generated/openapi.ts"
  },
  "bsb": {
    "hooks": {
      "afterSchemas": "generate-api-client"
    }
  }
}

Your script can read the generated schemas from src/.bsb/schemas/*.json and output TypeScript files that will be compiled in the next step.

Pre-compilation transforms

{
  "scripts": {
    "process-templates": "vite build --config templates.vite.config.ts"
  },
  "bsb": {
    "hooks": {
      "beforeCompile": "process-templates"
    }
  }
}

Post-build packaging

{
  "scripts": {
    "bundle-worker": "esbuild lib/worker.js --bundle --outfile=dist/worker.js",
    "generate-checksums": "sha256sum dist/worker.js"
  },
  "bsb": {
    "hooks": {
      "afterBuild": ["bundle-worker", "generate-checksums"]
    }
  }
}

Dev environment setup

{
  "scripts": {
    "seed-db": "prisma db seed",
    "start-deps": "docker compose up -d redis postgres"
  },
  "bsb": {
    "hooks": {
      "beforeDev": ["start-deps", "seed-db"]
    }
  }
}

Why not npm lifecycle hooks?

npm supports prebuild and postbuild scripts, but these only fire when you run npm run build. In dev mode, BSB calls its build pipeline functions directly -- npm lifecycle hooks never fire. BSB build hooks work consistently in both build and dev modes.

BSB hooks also give you finer-grained control. npm only has "before build" and "after build". BSB hooks let you target specific points like "after schema generation but before TypeScript compilation" -- which is exactly where custom codegen needs to run.

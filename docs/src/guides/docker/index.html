Docker Images

Base Docker image tags, startup behavior, and container environment variables. Production deployment recipes live in the Deploy section; language-specific image extension lives in each language guide.

Image Tags

Forgejo is the primary image registry. Docker Hub is a best-effort mirror and can lag if Docker Hub rate limits or rejects a push.

code.bettercorp.dev/bettercorp/service-base:node
code.bettercorp.dev/bettercorp/service-base:node-latest
code.bettercorp.dev/bettercorp/service-base:node-9.6.46
Use runtime-first tags. Use node, node-latest, or node-<version>. Do not use tags like 9-node.

Other runtime tags follow the same pattern: go, go-latest, python, python-latest, dotnet, and version-specific variants.

Node Image Contents

The official Node.js image runs BSB from /home/bsb/node_modules/@bsb/base. Core plugins resolve from that package, not from a copied local /home/bsb/lib tree.

  • @bsb/base is installed in /home/bsb/node_modules.
  • @bsb/base provides config-default, config-env, events-default, and observable-default.
  • @bsb/config-vault is bundled in the official Node image.
  • The image includes npm run start, npm run debug, and npm run plugins for shell/debug use.
  • The container runs as an unprivileged Node user after entrypoint preparation.

The same image can run plugin watcher mode by setting BSB_PLUGIN_WATCHER=true. Watcher mode syncs plugins and does not start BSB.

Default Environment

The Node image already sets these defaults. Dockerfiles that extend the image should not repeat them unless intentionally overriding behavior.

NODE_ENV=production
BSB_CONTAINER=true
BSB_LIVE=true
BSB_PLUGIN_DIRS=/mnt/plugins

Entrypoint Flow

The Node entrypoint does a small amount of container preparation before BSB starts:

  1. Create and fix ownership/permissions for BSB_WRITABLE_PATHS.
  2. Install packages listed in BSB_PLUGINS, if any.
  3. Print package/plugin discovery when BSB_SHOW_PACKAGES=true.
  4. Start BSB through @bsb/base.

Environment Reference

These variables control the container and BSB startup. See Production Deployment for concrete deployment examples.

Plugin Loading

  • BSB_PLUGIN_DIRS: comma-separated plugin directories. The first path is also the startup install target.
  • BSB_PLUGIN_DIR, BSB_PLUGINS_DIR: legacy aliases for plugin directories.
  • BSB_PLUGINS: comma-separated npm packages to install before startup. Supports no selector, major, minor, and exact selectors. @latest is rejected; omit the selector for npm latest.
  • BSB_PLUGIN_UPDATE=true: reinstall/refresh requested startup plugins.
  • BSB_PLUGIN_WATCHER=true: run plugin watcher mode instead of starting BSB.
  • BSB_PLUGIN_WATCH_INTERVAL_SECONDS: watcher image sync interval, defaulting to one hour.
  • BSB_PLUGIN_WATCH_ONCE=true: run one watcher sync and exit.
  • BSB_SHOW_PACKAGES=true: print package, plugin, type, version, path, metadata, and config-reference snippets before startup.

Config Selection

  • BSB_CONFIG_PLUGIN: active config plugin id. Common values: config-default, config-env, config-vault.
  • BSB_CONFIG_PLUGIN_PACKAGE: package containing the config plugin. Use @bsb/base for config-default and config-env; use @bsb/config-vault for config-vault.
  • BSB_CONFIG_FILE: config file path for config-default.
  • BSB_CONFIG_JSON: full JSON config for config-env.
  • BSB_PROFILE: config profile name, defaulting to default.

Config Plugin Schema Values

Config plugin fields are also read from env using their exact schema key names. They are case-sensitive.

  • Vault runtime keys: vaultUrl, apiKeyId, apiSecret.
  • Vault optional keys: timeoutMs, allowInsecureHttp.

Runtime State

  • BSB_WRITABLE_PATHS: comma-separated files/directories to create and make writable for runtime state, for example /data.
  • BSB_DEBUG=true: enable production debug logging.

Plugin Search Report

Inside an image or running container, print what BSB can discover with:

npm run plugins

In a derived Dockerfile, run the underlying report after copying or installing plugins:

RUN node /home/bsb/node_modules/@bsb/base/lib/scripts/list-plugin-search-paths.js

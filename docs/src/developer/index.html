Developer Guide

Deep-dive guidance for building, testing, deploying, and operating BSB services.

End-to-End Tutorial

  1. Scaffold a plugin project (see language guide)
  2. Implement events and handlers
  3. Configure default plugins (config/events/logging/metrics)
  4. Run locally with Docker Compose (mount plugin repo)
  5. Publish artifacts to plugin repository
  6. Deploy container with mounted repository

Plugin Lifecycle

Hooks

  • constructor(config): set up fields; no async work
  • init(trace): prepare dependencies, read config, register event listeners/handlers
  • run(trace): start processing and servicing events
  • dispose(): graceful shutdown and cleanup of resources

Enable/Disable & Order

  • Disabled plugins are skipped but logged
  • Dependencies determine initialization order
  • Use typed configuration validation

Event Design Guide

Conventions

  • Names: domain.action, kebab/snake as needed
  • Semantic payloads with explicit types
  • Include correlation IDs and timestamps

Reliability

  • Prefer idempotent handlers
  • Use dead-letter queues/topics if your adapter supports them
  • Retries/backoff/circuit breaking are app-level: choose libraries/patterns that fit your needs

Troubleshooting

  • Plugin not discovered: verify repo mount and naming
  • Config errors: validate schema and profile selection

Performance & Scaling

  • Tune concurrency per handler
  • Batch where adapters support it
  • Design for backpressure; implement shedding in your service if required

Security Hardening

  • Signed plugin artifacts and checksums
  • Secrets via config plugins; avoid env/plaintext
  • RBAC for config sources

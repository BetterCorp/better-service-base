Getting Started with BSB

Everything you need to know to build your first event-driven microservice with Better Service Base.

What is BSB?

BSB (Better Service Base) is an event-bus based microservice framework designed for simplicity, flexibility, and expandability. In production, BSB runs as a standalone Docker container that loads and runs plugins. Installing BSB as a dependency is only needed for contributors or when developing plugins locally. Plugins are published separately and provided to the container via a mounted plugin repository or optional on-boot installs.

🧠 Core Concepts

📡 Event-Bus Architecture

Services communicate through events, enabling loose coupling and reactive patterns. Services emit and listen to events without direct dependencies.

🔌 Plugin System

Services are defined as plugins in the {src|lib}/plugins/ directory structure. Each service is self-contained but exposes functions through typed Events interfaces. Plugins are discovered and loaded automatically.

📦 NPM Dependency

BSB is installed as a dependency (via npm install @bettercorp/service-base) and automatically discovers plugins in the {src|lib}/plugins/ directory. No manual wiring or complex configuration required.

🏗️ Flexible Deployment

Same codebase runs as monolith, tightly coupled services, microservices, or distributed edge nodes - you choose the deployment pattern.

🏗️ Architecture Patterns

BSB adapts to your deployment needs. Here are the main patterns:

🏢 Monolith

Single container with all services running together. Perfect for development and small applications.

┌─────────────────────┐ │ Single Container │ │ ┌───┐ ┌───┐ ┌───┐ │ │ │ A │ │ B │ │ C │ │ │ └───┘ └───┘ └───┘ │ │ Event Bus │ └─────────────────────┘

🔗 Tightly Coupled

Multiple services in same container, sharing event bus. Good for related services that scale together.

┌─────────────────────┐ │ Shared Container │ │ ┌─────┐ ┌─────┐ │ │ │ A │ │ B │ │ │ │ │ │ │ │ │ └─────┘ └─────┘ │ │ Shared Bus │ └─────────────────────┘

🚀 Microservices

Each service in its own container, individually scalable. Event bus connects distributed services.

┌─────┐ ┌─────┐ ┌─────┐ │ A │ │ B │ │ C │ │ │ │ x3 │ │ │ └─────┘ └─────┘ └─────┘ │ │ │ └────────┼────────┘ Event Bus

🌍 Edge/Global

Services distributed across multiple locations, connected over internet. Perfect for CDN-like architectures.

US-East EU-West ┌─────┐ ┌─────┐ │ A │ ─── │ B │ └─────┘ └─────┘ │ │ └─── Bus ───┘ │ ┌─────┐ │ C │ Asia └─────┘

Plugin System

Discover and use plugins to extend BSB functionality

Plugin Architecture

BSB's plugin system allows you to easily extend functionality for configuration management, event handling, logging, metrics collection, and custom services. In production, plugins are provided to the BSB container as prebuilt packages under a mounted plugin repository (BSB_PLUGIN_DIR). During development, you can structure plugins under {src|lib}/plugins/ in your plugin project for local testing. Plugins are automatically discovered and loaded based on your configuration.

🛠️ Developing Custom Plugins

Create your own plugins to extend BSB functionality. For production delivery, publish your plugin as an npm package or ship a prebuilt folder under the mounted repository structure used by BSB_PLUGIN_DIR:

/mnt/plugins/
  @org/your-plugin/
    latest/
      package.json
      lib/plugins/service-your-plugin/index.js
    1.0.0/
      package.json
      lib/plugins/service-your-plugin/index.js

Plugin Structure

plugins/my-custom-plugin/
├── package.json      # Plugin metadata
├── index.js         # Main plugin class
└── config.schema.js # Configuration schema (optional)

Basic Plugin Template

// plugins/my-custom-plugin/package.json
{
  "name": "my-custom-plugin",
  "version": "1.0.0",
  "main": "index.js",
  "bsb": {
    "type": "service",
    "name": "my-custom"
  }
}

// plugins/my-custom-plugin/index.js
const { BSBService } = require('@bettercorp/service-base');

class MyCustomPlugin extends BSBService {
  async onStart() {
    this.logger.info('My custom plugin starting...');
    
    // Plugin initialization logic
    await this.initialize();
    
    // Set up event listeners
    await this.onEvent('user.created', this.handleUserCreated.bind(this));
  }
  
  async initialize() {
    // Your plugin logic here
  }
  
  async handleUserCreated(data) {
    this.logger.info('Handling user created event', data);
    // Service communicates through the event bus
    await this.emitEvent('notification.send', {
      type: 'welcome',
      userId: data.id
    });
  }
  
  async onStop() {
    this.logger.info('My custom plugin stopping...');
    // Cleanup logic
  }
}

module.exports = MyCustomPlugin;

Plugin Discovery

BSB automatically discovers plugins by:

  • Looking under the mounted BSB_PLUGIN_DIR path for lib/plugins/* folders (preferred in production)
  • Scanning the {src|lib}/plugins/ directory structure in local plugin projects (development)
  • Reading package.json files for BSB metadata
  • Loading plugins based on your configuration
  • Initializing plugins in dependency order
  • Services can run together or separately based on deployment needs

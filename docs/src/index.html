How BSB Works

BSB is installed as a dependency in your project to provide types, functions, and framework foundation. Your services are defined as plugins in a specific structure where each service is self-contained but can expose functions through typed event interfaces.

🏗️ Deployment Flexibility

BSB services can run in various configurations depending on your needs - the event bus handles all communication.

Single Service

One service per container with full isolation

Multi-Service

Multiple services in same container

Distributed

Services across multiple containers

Global Edge

Worldwide distribution at edge locations

SaaS

Services distributed across multiple tenants

Hybrid

Mixed deployment patterns (SaaS + On-Prem)

On-Prem

Services running on-premises

Plugin-Based
